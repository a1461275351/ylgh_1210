# 1210 保税跨境电商关务管理系统 技术栈清单

> 归档日期:2026-07-06 ┃ 对应代码库:`C:\gong\1210`
> 说明:本清单为**原型阶段实际使用**的技术,并标注生产环境的推荐替换项。

## 一、总体架构

```
浏览器(SPA)
   │  HTTP / JSON(REST)
   ▼
Node.js(Express)后端  ──►  数据库(原型 SQLite / 生产 MySQL 8)
   │
   └──►  申报通道适配层(生产:单一窗口/金二报文 + 电子口岸加签;原型:回执模拟器)
```

- 前后端分离,后端同时托管前端静态资源(单进程,便于本地演示);
- 免构建:前端不需要 webpack/vite 打包,库文件由后端按本地路径直出。

## 二、后端

| 技术 | 版本 | 用途 | 生产建议 |
|------|------|------|----------|
| Node.js | v24.16.0 | 运行时 | 生产用 LTS(v22.x)更稳妥 |
| Express | 5.2.1 | Web 框架 / 路由 / 静态托管 | 沿用 |
| node:sqlite(`DatabaseSync`) | Node 内置 | 原型数据库驱动(同步 API) | 生产替换为 MySQL 驱动 |
| 原生 `fetch` | Node 内置 | 测试脚本发请求 | 沿用 |

- 后端代码组织:`server/src/index.js`(入口)、`db.js`(建表+种子)、`util.js`(响应/分页/单号/日志)、`routes/*`(每功能一个路由文件)。
- 约定:统一响应 `{ code, message, data }`;单号规则 `{前缀}{yyyyMMdd}{6位流水}`;所有写操作落 `sys_op_log` 审计。

## 三、前端

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue 3(global build) | 3.5.39 | 视图框架(免构建,`vue.global.prod.js`) |
| Element Plus | 2.14.2 | UI 组件库(表格/表单/弹窗/分页/菜单/页签) |
| @element-plus/icons-vue | 2.3.2 | 图标 |
| Element Plus 中文语言包 | 随库 | zh-cn 本地化 |

- 前端代码组织:`web/index.html`(壳)、`app.js`(布局:菜单树+多页签+页面注册器)、`styles.css`(主题)、`pages/*.js`(每功能一个页面组件,挂到 `window.PageRegistry`)。
- **无 CDN 依赖**:所有库文件由后端 `/libs/*` 路由从 `node_modules` 直出,内网/离线可运行。
- 生产可选升级:若需工程化(TS、组件单测、按需加载),改为 Vite + Vue SFC 构建。

## 四、数据库

| 环境 | 技术 | 文件 |
|------|------|------|
| 原型 | SQLite(WAL 模式) | `server/data/app.db`(运行时生成) |
| 生产 | MySQL 8.0(InnoDB / utf8mb4) | `db/schema.mysql.sql` |

- 原型 SQLite 与生产 MySQL DDL **字段级等价**,按功能任务递增建表;
- 金额 `DECIMAL(18,4)`、数量 `DECIMAL(18,5)`(海关申报精度);
- 个人信息(身份证号)约定应用层 AES 加密后入库,页面脱敏展示。

## 六、报文引擎(T19 已落地)

`server/src/ceb/`:自研零依赖 XML 组装(`xml.js`)、CEB621 清单报文(`messages.js`)、综合税试算(`tax.js`)、加签插槽(`sign.js`,原型 SHA-256 mock / 生产电子口岸卡)、通道适配 + 回执模拟器(`channels.js`)、申报编排(`engine.js`)。通道双轨 DIRECT/THIRD/SIMULATOR 可配置切换。
