# 1210 关务系统 · 本地测试清单

> 用途:在本地把系统测透。分两部分——**A 自动化测试**(1 分钟出结论)、**B 手工全流程测试**(照着点,逐屏核对)。
> 建议顺序:先跑 A 确认后端全绿,再做 B 在界面上把 1210 完整业务走一遍。

---

## A. 自动化测试(必做,约 1 分钟)

在 `server` 目录执行(或双击根目录 `run-tests.bat`):

```bash
npm run test:fresh
```

**预期结果:**
```
套件:9   通过用例:146   失败用例:0
✔ 全部通过(146 个用例)
```

覆盖范围(9 个套件 / 146 用例):

| 套件 | 用例 | 覆盖 |
|------|------|------|
| 客户管理 | 15 | 增删改查、分页、校验、唯一性 |
| 合同管理 | 11 | 合同维护、关联客户、有效期 |
| 接口平台/报文引擎 | 16 | CEB621组装、加签、通道、放行/退单回执、报文日志、重发 |
| 商品备案 | 19 | 备案流程、正面清单校验、HS税率、账册绑定、驳回 |
| 电子账册 | 17 | 备案流程、表体、台账、变更单核增 |
| 三单+清单 | 17 | 订单接入、6 道申报前校验、生成清单、放行、三单对碰 |
| 1210进口闭环 | 15 | 备货入区核增 → 零售放行 → 出区核减,账册全程一致 |
| 税费+退货 | 19 | 税单生成、担保占用/释放、退货核增+冲减+额度恢复 |
| WMS+报核 | 17 | 三账比对差异检出、盘点调账、报核核销流程 |

> 任一用例失败,脚本会打印失败套件与具体断言,便于定位。

## B. 手工全流程测试(推荐,约 15 分钟)

**准备:** 在 `server` 目录先重置为干净数据 → 启动:
```bash
npm run reset && npm start
```
浏览器打开 http://localhost:3010。

> 干净种子数据:1 个自营主体、4 个已备案商品、1 本账册 `T901625A00100`(初始库存:奶粉1000/维C 800/手袋50/护肤600)、担保额度 100 万。

下面按 1210 真实业务顺序走。每步给出【操作】和【预期】,逐项打勾 ☐。

### 阶段 1 · 备案核对(确认基础数据就位)

☐ **客户管理 → 客户档案**:能看到 1 条"自营"主体,备案类型为电商企业。
☐ **商品备案 → 商品备案资料库**:4 个商品,状态均为"备案通过",正面清单列显示"在册"(手袋等)。
☐ **商品备案 → HS税率库**:5 条 HS,其中 `9990001000` 显示"不在册"。
☐ **保税仓/物流账册 → 账册备案**:`T901625A00100` 状态"审批通过"。
☐ **保税仓/物流账册 → 账册台账**:选中账册,看到 4 个序号的初始结余(1000/800/50/600)。

### 阶段 2 · 备货进口入区(让货合法进保税仓)

☐ **报关单管理 → 报关单**:点【生成备货进口报关单】→ 列表新增 1 条,监管方式 1210,状态"录入"。
☐ 该行点【申报】→ 状态变"已申报";再点【放行】→ 状态"放行"。
☐ 点【生成核注】→ 提示"已生成入区核注清单"。
☐ **核注清单管理 → 核注清单**:筛选方向"一线入区",找到新单,状态"录入"。
☐ 点【申报】→"已申报";再点【审批】→ 弹出"审批通过,账册已核增"。
☐ 回到 **账册台账**:奶粉/维C/手袋结余各 **+500**(如奶粉 1000→1500)。✅ 核增生效
☐ 该核注行点【生成核放】→ 提示已生成核放单。
☐ **核放单管理 → 核放单**:找到"一线入区"核放单 → 【申报】→【过卡验放】→ 状态"已过卡入区"。
☐ **WMS协同 → 入库单**:点【同步入库单】→ 出现对应入库单,状态"理货完成"。

### 阶段 3 · 零售清关(消费者下单到放行)

☐ **三单数据中心 → 订单管理**:样例选"正常订单",点【接入样例订单】→ 新增订单,校验列显示"✔ 通过",状态"已校验"。
☐ 再接入一单"超单次限值"→ 状态"校验失败",校验列红字"订单金额…超单次限值 5000 元"。✅ 前置校验拦截
☐ 正常订单行点【生成清单】→ 状态变"已生成清单"。
☐ **清单管理 → 申报清单**:找到该清单,点【申报推送】→ 弹出"海关放行,清单编号 I…"。
☐ 点【详情/报文】→ 报文链路有"发送 CEB621"和"接收 CEB622",点开可看 XML 原文。✅ 报文引擎
☐ 回订单管理,该订单【详情】→ 分别【登记支付单】【登记运单】。
☐ **三单数据中心 → 三单对碰监控**:该订单"订/支/运/清"四栏全绿✔,状态"齐全"。
☐ **税费管理 → 税单管理**:出现该订单税单,状态"待缴",综合税=清单税额。
☐ **税费管理 → 担保额度**:已占用金额增加,流水多一条"清单放行占用"。✅ 税费+担保联动

### 阶段 4 · 出区发货(货发给消费者)

☐ **核注清单管理 → 核注清单**:点【生成出区核注(放行清单汇总)】→ 勾选放行清单 → 生成出区核注(方向"二线出区")。
☐ 该单【申报】→【审批】→ 提示"账册已核减"。
☐ **账册台账**:对应序号结余 **减少**(如奶粉 -2)。✅ 出区核减
☐ 该核注行【生成核放】→ **核放单管理** 找到"二线出区"核放单 →【申报】→【过卡验放】→ 提示"货物已过卡出区,N 个订单已出库"。
☐ 回 **订单管理**:该订单状态变"已出库"。✅ 全链路闭环
☐ **WMS协同 → 出库单**:点【同步出库单】→ 出现出库单,可【推进】到已发货。

### 阶段 5 · 退货(逆向闭环)

☐ **退货管理 → 退货单**:点【发起退货申请】→ 弹窗列出可退清单(显示放行天数、是否在 30 日内)→ 选一张放行清单、填原因 → 提交。
☐ 列表新增退货单,状态"已申请";对应订单状态变"退货中"。
☐ 该退货行【申报退货清单】→ 状态"海关通过"。
☐ 点【入区理货】→ 提示"账册已核增、税款已冲减"。
☐ 核对三处联动:**账册台账**对应序号结余回升;**税单管理**该税单状态"已冲减";**担保额度**已占用回落。✅ 退货三联动

### 阶段 6 · 对账合规(海关稽核视角)

☐ **WMS协同 → 三账比对**:点【运行三账比对】→ 表格显示每序号 关务账册/WMS实物/客户系统 三栏,结果"相符"。
☐ 某行点【调实物】,把实物改成与账册不同的值 → 再【运行三账比对】→ 该行变红、结果"差异",顶部"差异 1 项"。✅ 账实差异可检出
☐ **报核核销 → 盘点管理**:点【生成盘点单】→【盘点明细】看账面 vs 实盘盈亏 →【确认】→【调整账册】→ 账册按盈亏调平。
☐ **报核核销 → 报核管理**:点【生成报核单】→【差异表】查看账面/实际 → 【申报】→【通过】→【结案】,状态走到"核销结案"。

### 阶段 7 · 接口平台核对(可选)

☐ **系统管理 → 通道配置**:看到 SIMULATOR(默认)/DIRECT(直连,待联调)/THIRD 三条通道。
☐ **系统管理 → 报文日志**:能查到所有 CEB621 发送、CEB622 接收记录,可看原文、失败可重发。

## 完成标准

- ✅ A 自动化:146 用例全通过;
- ✅ B 手工:阶段 1–6 全部打勾,尤其验证 **账册库存随核注核增/核减真实变动**、**税费/担保/退货三联动**、**三账比对能检出差异**。

达成即说明:**1210 完整进口业务闭环 + 财务合规 + 对账稽核**在本地跑通。
