## 1. 项目背景与目标

### 1.1 背景

公司在综合保税区内**以自营为主**开展 1210 保税备货进口业务,同时具备对外提供清关、仓储服务的能力(服务商模式)。需要一套关务管理系统,对内管理客户、合同、商品备案、电子账册、库存核减,对外对接:

- **国际贸易单一窗口 / 跨境电商通关服务平台**(三单推送、清单申报);
- **海关跨境电商进口统一版系统**(清单审放、税费);
- **金关二期(金二)特殊区域管理系统**(账册、核注清单、核放单、报核);
- **电商平台 / 客户 ERP·OMS**(订单接入);
- **保税仓 WMS**(入库、出库、库存)。

### 1.2 建设目标

1. 打通"订单 → 三单对碰 → 清单申报 → 账册核注 → 核放出区 → 发货"全链路,实现逐单自动化清关;
2. 电子账册(物流账册/H账册)全生命周期线上管理:备案、变更、初审、复审、核销;
3. 关务数据与仓储库存实时一致,账货相符,可随时应对海关稽核;
4. 以自营业务为主线设计,同时按多客户(多电商企业)、多店铺、多平台的多租户结构建模,保留服务商运营能力;
5. 申报通道双轨建设:自建直连(单一窗口/金二标准报文 + 电子口岸加签)与第三方通关服务商通道均支持,可配置切换、互为灾备;
6. 全流程留痕,单证状态可追溯,统计报表齐全。

### 1.3 参考系统

界面与模块组织参考现有"关务管理平台"(客户管理、合同管理、结关管理、加工贸易账册、保税仓/物流账册、核注清单、报关单、核放单、报核、核销、综合查询),在此基础上补充跨境电商 1210 特有模块。

## 3. 总体业务流程(1210 进口)

```
① 备案阶段
   企业备案(电商企业/平台/支付/物流) → 商品备案(HS归类/要素/正面清单校验)
   → 电子账册备案(金二) → 账册审批通过

② 一线入区(批量备货)
   境外发货 → 进境报关(核注清单+报关单/两步申报) → 卡口核放入区
   → WMS 入库理货 → 账册库存增加(核注核增)

③ 零售订单清关(逐单)
   消费者下单 → 平台推订单 / 支付企业推支付单 / 物流企业推运单
   → 系统接收订单生成清单 → 申报清单(单一窗口/通关服务平台)
   → 统一版三单对碰 → 征税(担保额度扣减) → 清单放行

④ 二线出区(发货)
   放行清单 → 汇总生成核注清单(核减账册) → 核放单 → 卡口验放出区
   → WMS 拣选打包交快递 → 物流妥投

⑤ 汇总与核销
   清单汇总(集报)→ 生成报关单 → 账册报核 → 核销平账
   税款汇总缴纳、担保额度恢复

⑥ 退货(逆向)
   消费者退货 → 退货申请 → 30天内退回区内 → 退货清单申报
   → 入区理货 → 账册核增 → 税款冲减/额度恢复
```

## 4. 功能需求

### 4.1 基础与备案管理

#### 4.1.1 客户管理
- 客户档案:企业名称、统一社会信用代码、海关注册编码(10位)、跨境电商备案类型(电商企业/平台/支付/物流/仓储)、联系人、结算方式;
- 客户关系维护/查询(参考现有平台):客户与账册、店铺、商品库的关联关系;
- 客户资质证照管理(营业执照、海关备案回执等附件),到期提醒。

#### 4.1.2 合同/协议管理
- 服务合同(仓储、清关、配送)维护与查询,关联客户与计费规则;
- 合同有效期、额度提醒。

#### 4.1.3 商品备案资料库(核心)
- 商品档案:商品编码(SKU)、品名、HS 编码、申报要素、规格型号、原产国、品牌、条码、计量单位(法定/申报/销售)、净重毛重、单价币制;
- **正面清单校验**:维护跨境电商零售进口商品正面清单库,备案时自动校验 HS 是否在清单内;
- **税率库**:HS 对应增值税率、消费税率,自动计算综合税率;
- 归类辅助:同 HS 历史归类参考、要素模板;
- 商品与账册料号的对应关系(账册备案序号绑定),支持一品多号/多品一号的对应关系维护(参考现有平台"进出境关别对应关系"类配置);
- 备案流程:录入 → 初审 → 复审 → 推送账册备案/变更;
- 商品资料批量导入(Excel)、版本历史。

#### 4.1.4 企业参数与政策参数配置
- 监管方式、运输方式、关别、币制、国别、计量单位等海关标准代码表(可更新);
- 个人年度额度、单次限值、综合税折扣系数等政策参数**可配置**(政策调整时不改代码)。

### 4.2 电子账册管理(保税仓/物流账册)

对应现有平台"保税仓/物流账册"模块,通过金二报文或单一窗口界面辅助操作:

| 功能 | 说明 |
|------|------|
| 账册备案 | 新建物流账册(T)/加工账册(H)备案数据,商品表体维护,生成备案报文 |
| 账册变更 | 表头/表体变更(增加商品、修改申报要素等),变更单流水管理 |
| 账册初审 | 内部初审岗审核,审核意见留痕 |
| 账册复审 | 复审通过后允许申报;驳回退回修改 |
| 账册申报与回执 | 推送金二/单一窗口,接收审批回执,状态跟踪(如"变更金二审批通过") |
| 账册台账 | 备案序号维度的进、出、存、结余数量金额实时台账 |
| 账册有效期管理 | 结束有效期预警(如截图中 2026-12-31),到期提醒延期 |

**列表字段参考现有平台**:企业内部编号、平台预录入号、账册编号、账册类型、单证状态、经营单位代码/名称、结束有效期、录入日期。

### 4.3 三单数据中心(1210 特有,核心)

#### 4.3.1 订单接入
- API 接口(REST/JSON)+ Excel 导入两种方式接收电商平台/客户 ERP 订单;
- 订单要素:订单号、平台代码、企业代码、购买人(姓名、身份证号)、商品明细(SKU、数量、单价)、运费、税款、实际支付金额;
- 订单状态机:已接收 → 已校验 → 已生成清单 → 申报中 → 放行 → 出库 → 签收 / 异常;
- **前置校验**(申报前拦截,减少海关退单):
  - 身份证号规则校验、订购人与支付人一致性;
  - 商品是否已备案、是否在正面清单、账册是否有余量;
  - 单次 5000 元限值、个人年度额度校验(对接总署额度查询接口或本地累计);
  - 价格异常预警(与备案价偏离阈值)。

#### 4.3.2 支付单管理
- 接收/登记支付单数据(支付企业推送回执状态),与订单关联比对;
- 支付单与订单金额不一致预警。

#### 4.3.3 运单管理
- 与物流企业对接获取运单号,或由 WMS 打单后回传;
- 运单状态跟踪(揽收、出区、派送、妥投)。

#### 4.3.4 三单对碰监控
- 展示每笔订单的订单/支付/运单/清单四单状态矩阵,缺单、超时未对碰自动预警;
- 对碰失败原因归类统计(金额不一致、身份信息错误、缺支付单等)。

### 4.4 清单管理(1210 特有,核心)

#### 4.4.1 进口申报清单
- 由订单自动生成清单(含账册备案序号、征免方式等申报要素),支持人工修改补录;
- 制单 → 初审 → 复审 → 申报(可配置跳过人工审核,直接自动申报);
- 报文推送单一窗口/通关服务平台(清单申报 CEB621/CEB622 回执等标准报文,电子口岸卡加签;具体报文号以海关最新统一版规范为准);
- 回执解析与状态跟踪:已申报、海关退单(附退单原因)、放行、查验;
- 退单处理工作台:按退单原因批量修改、重报;
- 撤销单、删改单管理。

#### 4.4.2 集报清单 / 汇总申报
- 按周期(月度)汇总已放行清单,生成汇总报关单数据;
- 汇总规则:同账册、同 HS、同币制归并;
- 与报关单管理模块联动。

#### 4.4.3 税费管理
- 逐单综合税计算(增值税、消费税 ×70%),与海关回执税款核对;
- **担保额度管理**:总担保额度、已占用、已释放实时台账,余额不足预警;
- 电子税单接收、汇总缴税记录、与客户结算的税金账单导出。

> 业务范围已确认为仅 1210 进口,出口清单(1210 出口/9610)不在本项目范围内。

### 4.5 核注清单管理

- 由放行清单汇总(出区方向)或进境备货(入区方向)生成核注清单;
- 表头表体编辑、料号与账册序号对应校验、余量校验;
- 制单 → 初审 → 复审 → 申报金二 → 回执状态跟踪(预审批通过/审批通过/退单);
- 核注清单与清单、报关单、核放单的关联关系可查(单证链路追溯)。

### 4.6 核放单管理

- 依据核注清单生成核放单(一线入区、二线出区、区间流转、分送集报);
- 车辆信息、集装箱/托盘信息、IC 卡信息维护;
- 申报金二、卡口验放状态回执;
- 卡口异常(过卡失败)处理登记。

### 4.7 报关单管理

- 一线进境备货报关单(监管方式 1210)预录入、申报、回执、放行状态管理;
- 集报汇总报关单管理;
- 报关单表体申报单价维护(参考现有平台功能);
- 报关单与核注清单、清单的关联追溯;
- 单证归档:报关单、提运单、发票、箱单、原产地证等附件管理。

### 4.8 报核与核销管理

- 账册周期报核:生成报核报文、差异表(账面 vs 实际);
- 盘点管理:与 WMS 盘点结果比对,盘盈盘亏处理(补税/核增核减申请);
- 核销结案记录、历史核销档案查询。

### 4.9 退货管理(1210 特有)

- 退货申请接收(平台/客户发起),校验:原清单放行后 30 日内(可配置)、原状退回;
- 生成退货申报清单(CEB625 等),申报与回执跟踪;
- 退货入区理货结果登记(WMS 回传),账册核增;
- 税款冲减与个人年度额度恢复跟踪;
- 拒收/退货不入区(退运境外、放弃)流程分支。

### 4.10 库存与 WMS 协同

- 接收 WMS 入库单、出库单、库存快照;
- **三账比对**:关务账册库存 vs WMS 实物库存 vs 客户系统库存,差异报表;
- 出库指令下发:清单放行后自动向 WMS 下发拣货指令(接口);
- 效期、批次维度库存查询(如需)。

### 4.11 ERP 数据中心 / 接口平台

- **申报通道适配层(双轨,均建设)**:
  - 自建直连通道:单一窗口/跨境电商通关服务平台 CEB 标准报文、金二账册核注核放报文,含电子口岸 IC 卡/Ukey 加签服务;
  - 第三方通道:对接报关行/通关服务商的申报接口,报文格式转换适配;
  - 通道按单证类型、客户维度可配置切换,一条通道故障时可人工切换到另一条(灾备);
- 统一管理对外接口:客户 ERP/OMS、电商平台、WMS、单一窗口、金二;
- 报文收发日志、失败重发、报文原文查询(排查问题必备);
- 接口账号与密钥管理、IC 卡/Ukey 加签服务状态监控;
- 数据字典与代码表同步。

### 4.12 综合查询与统计

- 业务票数统计(按客户、平台、日期、状态多维度);
- 清单申报量/放行率/退单率日报、月报;
- 账册余量表、进出存报表;
- 税金统计、担保额度使用统计;
- 单证全链路查询:输入订单号/运单号/清单号,展示全流程节点时间轴;
- 报表导出 Excel,支持定时邮件推送(可选)。

### 4.13 系统管理

- 组织机构、用户、角色、权限(菜单级 + 数据级:按客户/账册隔离);
- 多公司主体支持(如集团下多个区内企业,参考现有平台 companyType=group);
- 操作日志、申报日志审计;
- 流程配置:初审/复审环节可按单证类型启用/停用;
- 消息中心:退单、额度不足、账册到期、对碰超时等站内+短信/企微提醒。

## 5. 外部接口清单

| 接口对象 | 方向 | 内容 | 方式 |
|----------|------|------|------|
| 单一窗口/跨境电商通关服务平台 | 双向 | 订单(CEB311/312)、清单申报(CEB621/622)、退货、撤销、回执 | 标准 XML 报文 + 电子口岸加签(客户端代理/接口对接,待确认) |
| 海关金关二期 | 双向 | 账册备案变更、核注清单、核放单、报核 | 金二数据交换报文(待确认对接通道) |
| 电商平台/客户 ERP | 入 | 订单、退货申请;出:清关状态回传 | REST API / Excel |
| 支付企业 | 入 | 支付单状态(通常支付企业直推海关,本系统记录状态) | API/对账文件 |
| 物流企业/快递 | 双向 | 运单号获取、轨迹、妥投状态 | API |
| WMS | 双向 | 入库单、出库指令、出库回传、库存、盘点 | API/MQ |

> 注:自营模式下,**订单与清单由本系统推送**;支付单、运单由支付企业、物流企业各自向海关推送,本系统记录并监控其对碰状态。服务商模式下受托代推订单时同样走本系统订单报文通道。

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 大促场景:清单申报峰值 ≥ 10 万单/日,单笔从接单到申报 ≤ 1 分钟;报文异步队列化处理 |
| 可用性 | 核心申报链路 7×24,可用性 ≥ 99.5%;报文失败自动重试 + 人工干预台 |
| 安全 | 购买人身份证号等个人信息加密存储、脱敏展示;传输 HTTPS;等保二级(建议) |
| 留痕 | 所有申报报文原文、回执原文保存 ≥ 3 年,满足海关稽核 |
| 可配置 | 税率、额度、限值、审核流程、归并规则均参数化 |
