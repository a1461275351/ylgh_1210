## 5. 关键业务闭环验证(实测)

```
备货进口入区(报关单→入区核注核增账册→核放过卡)
→ 零售清关(订单→三单校验→清单申报放行→税单+担保占用)
→ 出区发货(出区核注核减账册→核放过卡→订单已出库)
→ 退货(账册核增+税款冲减+额度释放)
→ 对账合规(三账比对差异检出→盘点调账→报核核销)
```
账册轨迹实测:期初 → 备货入区(+500)→ 出区发货(−N),核增核减由核注清单真实驱动。

## 6. 已知边界与上线实施事项(重要)

以下两项**不属于软件功能缺陷**,而是从"功能版"到"生产上线"必经的实施工作,均带**外部依赖**,需在具备条件后单独实施与验收:

### 6.1 切换生产数据库(SQLite → MySQL)
- 现状:原型使用内置 SQLite(单文件、零配置、开箱即用),已满足功能验收与演示;
- 已就绪:生产 MySQL 建表脚本 `db/schema.mysql.sql`(43 张表,结构与原型对齐);
- 待实施:需提供一台 MySQL 8 实例;将后端数据访问层从同步 SQLite 适配为 MySQL(异步驱动),并在该实例上联测;
- 说明:此项需真实 MySQL 环境方可执行与验收,详见《上线实施方案.md》。

### 6.2 对接真实单一窗口通道
- 现状:接口平台已内置**回执模拟器**,报文引擎(CEB621 组装/加签插槽/通道适配)全链路可跑通;
- 待实施(外部条件):海关跨境电商**测试环境接入信息**、**电子口岸 IC 卡/证书加签中间件**、企业**联调与验收**;
- 说明:真实通道对接依赖海关测试环境与资质联调,非软件单方可完成;架构已预留 DIRECT 直连通道,条件到位后替换模拟器即可,详见《上线实施方案.md》。
