# 1210 关务系统 · 部署迁移与本地测试指南

> 面向:在一台新电脑(Windows / Linux / Mac)上部署本系统并完成本地测试。
> 结论先行:**迁移非常轻**——拷贝项目文件夹 + 装 Node + 一条命令即可运行。

---

## 一、迁移到底有多方便

| 要拷的 | 大小 | 说明 |
|--------|------|------|
| 源代码(server/src、web、db、docs、脚本) | 约几百 KB | 真正的项目内容 |
| `node_modules` | **不用拷** | 到新机器 `npm install` 自动重建(93MB) |
| 数据库 `server/data/app.db` | 可拷可不拷 | 拷了带数据;不拷则首次启动自动建库+种子数据 |

- **无平台绑定**:Windows / Linux / Mac / 云服务器 / 内网专网都能跑;
- **无外网依赖**:前端库全部从本机加载(免 CDN),隔离网也能用;
- **唯一环境要求**:目标机器安装 **Node.js 20 或更高版本**(本机开发用 v24)。

---

## 二、迁移三步走

### 第 1 步:准备目标机器
安装 Node.js(LTS 版即可):https://nodejs.org
验证:命令行执行 `node -v`,能看到版本号(≥ v20)即可。

### 第 2 步:拷贝项目
把整个 `C:\gong\1210` 文件夹拷到目标机器任意位置(U 盘 / 压缩包 / 内网共享均可)。
> 拷之前可删掉 `server/node_modules` 减小体积(目标机器会重新安装)。

### 第 3 步:启动
- **Windows**:双击项目根目录的 **`start.bat`**
- **Linux/Mac**:执行 `bash start.sh`

首次启动会自动 `npm install`(需联网一次,约 1 分钟),随后打印:
```
1210 关务系统已启动,请用浏览器打开:http://localhost:3010
```
浏览器打开 http://localhost:3010 即可使用。

> 手动方式等价于:`cd server` → `npm install` → `npm start`

## 三、一键命令速查

在 `server` 目录下:

| 命令 | 作用 |
|------|------|
| `npm start` | 启动服务(http://localhost:3010) |
| `npm test` | 全量自动化测试(当前数据库) |
| `npm run test:fresh` | 全量测试(**先重置为干净数据库**,推荐) |
| `npm run reset` | 重置数据库(清空数据,下次启动重建种子) |

根目录下还有双击即用的:`start.bat`(启动)、`run-tests.bat`(全量测试)。

## 四、切换到生产数据库(MySQL)

原型用 SQLite(单文件、零配置),**正式上线建议换 MySQL 8**。迁移路径已备好:

1. 安装 MySQL 8,创建库并导入表结构:
   ```sql
   SOURCE db/schema.mysql.sql;
   ```
   (该脚本含全部 36 张表的建表语句、中文注释、索引和初始参数/代码表种子。)
2. 后端接入 MySQL 驱动(如 `mysql2`),把 `server/src/db.js` 的 SQLite 连接替换为 MySQL 连接池;因 DDL 与 SQLite 字段级等价,SQL 语句基本无需改动。
3. 配置连接信息(建议用环境变量:`DB_HOST/DB_USER/DB_PASS/DB_NAME`)。

> 说明:原型的 `db.js` 用了少量 SQLite 语法(`datetime('now','localtime')`、`ON CONFLICT`),迁 MySQL 时替换为 `NOW()`、`ON DUPLICATE KEY UPDATE` 即可,已在代码注释标注。

## 五、端口 / 配置

| 项 | 默认 | 如何改 |
|----|------|--------|
| 服务端口 | 3010 | 启动前设环境变量 `PORT`,如 `set PORT=8080`(Win)/ `PORT=8080 npm start`(Mac) |
| 测试端口 | 3020 | 测试脚本内部使用,自动隔离,不影响正在运行的服务 |
| 政策参数(限值/额度/税率折扣/退货天数/账册预警) | 见 `sys_param` 表 | 系统内"参数配置"或直接改表,无需改代码 |

## 六、常见问题

| 现象 | 原因 / 处理 |
|------|-------------|
| 双击 start.bat 闪退 | 未装 Node.js,或装后未重开命令行;装好 Node 再试 |
| `npm install` 失败 | 网络问题;可配置国内镜像 `npm config set registry https://registry.npmmirror.com` |
| 端口被占用 | 改用其它端口:`set PORT=8081 && npm start` |
| 页面样式错乱/空白 | 确认是从后端 3010 端口打开,而非直接双击 web/index.html |
| 测试报"数据库被占用" | 先停掉正在运行的服务再跑 `test:fresh`(测试会自己起独立服务) |
| 中文数据乱码 | 用系统内页面或 Node 脚本录入;不要用命令行 curl 提交中文(编码问题) |

## 七、本地测试怎么做

分两层,详见配套文档 **《本地测试清单.md》**:

1. **自动化测试**(1 分钟):`npm run test:fresh` —— 9 个套件、146 个用例,覆盖全部业务链路;
2. **手工 UI 冒烟测试**(15 分钟):按《本地测试清单.md》在浏览器里把 1210 完整进口流程走一遍,逐屏核对预期结果。
